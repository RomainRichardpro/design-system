## Accessibilité

Le composant `Checkbox` repose sur un `<input type="checkbox">` natif, ce qui garantit un comportement conforme aux standards sans nécessiter de configuration ARIA complexe.

<table width="100%">
  <thead>
    <tr>
      <th align="left">Critère</th>
      <th align="left">Détail</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Élément natif</strong></td>
      <td>Utilise <code>&lt;input type="checkbox"&gt;</code> — rôle <code>checkbox</code> implicite, compatible avec tous les lecteurs d'écran (NVDA, JAWS, VoiceOver) sans configuration supplémentaire.</td>
    </tr>
    <tr>
      <td><strong>Association label</strong></td>
      <td>Un <code>&lt;label htmlFor&gt;</code> est systématiquement associé à l'input. L'identifiant est auto-généré via <code>useId()</code> si le prop <code>id</code> n'est pas fourni — l'association est donc toujours garantie.</td>
    </tr>
    <tr>
      <td><strong>État indéterminé</strong></td>
      <td>La propriété DOM <code>.indeterminate</code> est appliquée via <code>useEffect</code> sur la ref de l'input (attribut non standard HTML, non supporté via JSX). Les lecteurs d'écran annoncent l'état via <code>aria-checked="mixed"</code>.</td>
    </tr>
    <tr>
      <td><strong>Label masqué</strong></td>
      <td>Quand <code>hideLabel</code> est actif, le texte du label est masqué visuellement via la classe <code>.srOnly</code> (position absolute, dimensions 1px) mais reste présent dans le DOM et accessible aux technologies d'assistance. Ne jamais passer une chaîne vide.</td>
    </tr>
    <tr>
      <td><strong>Focus visible</strong></td>
      <td>Halo appliqué via <code>:focus-visible</code> sur le container racine grâce au sélecteur <code>:has(.input:focus-visible)</code>. Couleur <code>var(--focus)</code>, épaisseur <code>var(--border-width-03)</code>. Conforme WCAG 2.1 AA critère 2.4.7.</td>
    </tr>
    <tr>
      <td><strong>Navigation clavier</strong></td>
      <td><code>Tab</code> pour atteindre la checkbox · <code>Space</code> pour basculer l'état. Comportement natif garanti par l'élément <code>&lt;input&gt;</code>.</td>
    </tr>
    <tr>
      <td><strong>État désactivé</strong></td>
      <td>L'attribut natif <code>disabled</code> est utilisé sur l'input — exclu du flux de focus, non interactif. Les états visuels (coché, indéterminé) sont préservés pour communiquer l'état courant.</td>
    </tr>
    <tr>
      <td><strong>Contraste</strong></td>
      <td>Ratio testé WCAG 2.1 AA (minimum 4.5:1) sur le texte du label, états default et disabled inclus.</td>
    </tr>
  </tbody>
</table>
