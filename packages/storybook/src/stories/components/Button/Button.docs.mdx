## Accessibilité

Le composant `Button` utilise un élément `<button>` natif, ce qui garantit une compatibilité maximale avec les technologies d'assistance sans configuration supplémentaire.

<table width="100%">
  <thead>
    <tr>
      <th align="left">Critère</th>
      <th align="left">Détail</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Élément natif</strong></td>
      <td>Utilise <code>&lt;button type="button"&gt;</code> — rôle <code>button</code> implicite, pas besoin de <code>role</code> ARIA explicite.</td>
    </tr>
    <tr>
      <td><strong>État chargement</strong></td>
      <td><code>aria-busy="true"</code> est ajouté quand <code>loading</code> est actif. Le contenu visible est remplacé par un spinner muet (<code>aria-hidden</code>). Le prop <code>loadingLabel</code> est rendu via une <code>span</code> visuellement masquée (<code>.srOnly</code>) et lu par les lecteurs d'écran à la place.</td>
    </tr>
    <tr>
      <td><strong>État désactivé</strong></td>
      <td>L'attribut natif <code>disabled</code> est utilisé. Le bouton est exclu du flux de focus (<code>tabIndex=-1</code> implicite) et les événements sont bloqués nativement par le navigateur.</td>
    </tr>
    <tr>
      <td><strong>Focus visible</strong></td>
      <td>Halo appliqué via <code>:focus-visible</code> uniquement — ne s'affiche pas lors d'un clic souris, uniquement lors de la navigation clavier. Couleur <code>var(--focus)</code>, épaisseur <code>var(--border-width-03)</code>, offset 4px. Conforme WCAG 2.1 AA critère 2.4.7.</td>
    </tr>
    <tr>
      <td><strong>Navigation clavier</strong></td>
      <td><code>Tab</code> pour atteindre le bouton · <code>Enter</code> ou <code>Space</code> pour déclencher l'action. Comportement natif garanti par l'élément <code>&lt;button&gt;</code>.</td>
    </tr>
    <tr>
      <td><strong>Contraste texte</strong></td>
      <td>Ratio testé WCAG 2.1 AA (minimum 4.5:1) sur <code>primary</code> et <code>secondary</code>, états default et disabled inclus.</td>
    </tr>
    <tr>
      <td><strong>Contenu minimal</strong></td>
      <td>Le bouton doit toujours avoir un libellé explicite via <code>children</code>. Éviter les libellés ambigus comme "OK" ou "Valider" — préférer "Enregistrer les modifications".</td>
    </tr>
  </tbody>
</table>
