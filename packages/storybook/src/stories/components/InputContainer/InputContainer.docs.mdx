## Accessibilité

`InputContainer` assemble plusieurs sous-composants avec des liaisons ARIA automatiques garantissant une expérience correcte pour les technologies d'assistance.

<table width="100%">
  <thead>
    <tr>
      <th align="left">Critère</th>
      <th align="left">Détail</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Association label / input</strong></td>
      <td>Le label est lié à l'input via <code>htmlFor</code> / <code>useId()</code> — navigation clavier et lecteurs d'écran corrects sans configuration supplémentaire.</td>
    </tr>
    <tr>
      <td><strong>Champ obligatoire</strong></td>
      <td><code>aria-required="true"</code> est posé sur l'input quand <code>isRequired={`{true}`}</code>. Un astérisque rouge est affiché visuellement après le label.</td>
    </tr>
    <tr>
      <td><strong>Message d'aide</strong></td>
      <td><code>aria-describedby</code> lie l'input au SupportingText quand <code>withSupportingText={`{true}`}</code> — le message est lu après le label par les lecteurs d'écran.</td>
    </tr>
    <tr>
      <td><strong>Statut Error</strong></td>
      <td><code>aria-invalid="true"</code> est propagé sur l'input. Le SupportingText passe en <code>role="alert"</code> pour être annoncé automatiquement sans action supplémentaire.</td>
    </tr>
    <tr>
      <td><strong>État Disabled</strong></td>
      <td>L'attribut natif <code>disabled</code> est utilisé sur l'input — exclu du flux de focus, non interactif. Le label et le SupportingText passent en couleur <code>--text-disabled</code>.</td>
    </tr>
    <tr>
      <td><strong>État Read-only</strong></td>
      <td><code>readOnly</code> est posé sur l'input — le champ reste focusable et lisible par les lecteurs d'écran, mais la saisie est bloquée.</td>
    </tr>
    <tr>
      <td><strong>Focus visible</strong></td>
      <td>Halo appliqué via <code>:focus-within</code> sur le wrapper du TextInput. Couleur <code>var(--focus)</code>, épaisseur <code>var(--border-width-02)</code>. Conforme WCAG 2.1 AA critère 2.4.7.</td>
    </tr>
  </tbody>
</table>
