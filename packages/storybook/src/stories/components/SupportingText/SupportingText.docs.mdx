## Accessibilité

<table width="100%">
  <thead>
    <tr>
      <th align="left">Critère</th>
      <th align="left">Détail</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Annonce automatique des erreurs</strong></td>
      <td><code>role="alert"</code> est posé <strong>uniquement</strong> quand <code>status="Error"</code> et <code>state !== "Disabled"</code>. Le message est annoncé automatiquement par les lecteurs d'écran dès son apparition ou sa modification.</td>
    </tr>
    <tr>
      <td><strong>Icône décorative</strong></td>
      <td>L'icône porte <code>aria-hidden="true"</code> — elle est purement décorative et ne doit pas transmettre d'information unique (la couleur et le texte suffisent).</td>
    </tr>
    <tr>
      <td><strong>Sémantique du texte</strong></td>
      <td>Le message est rendu dans un <code>&lt;p&gt;</code> — sémantique correcte, compatible avec tous les lecteurs d'écran.</td>
    </tr>
    <tr>
      <td><strong>Liaison avec l'input</strong></td>
      <td>Utilisé via <code>aria-describedby</code> depuis le champ parent — le message est lu après le label lors de la prise de focus sur l'input. Cette liaison est gérée automatiquement par <code>InputContainer</code>.</td>
    </tr>
    <tr>
      <td><strong>État Disabled</strong></td>
      <td>En état <code>Disabled</code>, <code>role="alert"</code> n'est pas posé même si <code>status="Error"</code> — un champ désactivé ne doit pas déclencher d'annonce.</td>
    </tr>
  </tbody>
</table>
